# s&box AMP Generic Module Template - Draft

Files included:

- `sbox.kvp`
- `sboxconfig.json`
- `sboxports.json`
- `sboxupdates.json`

Default launch line generated by AMP:

```bash
./sbox-server.exe +game "facepunch.sandbox" "facepunch.flatgrass" +hostname "s&box Server - Powered by AMP" +port 27015 +net_query_port 27016
```

## Notes

- This is a draft for personal use/testing in AMP.
- Linux requires the .NET Runtime. If the server fails with a .NET error, install the runtime requested by the error message.
- The template uses SteamCMD AppID `1892930` for the dedicated server and Steam AppID `590830` for display/Steam environment.
- Default ports are UDP 27015 for game traffic and UDP 27016 for query.
- If the public/staging branch changes, edit `sboxupdates.json` and add `"UpdateSourceVersion": "staging"` to the SteamCMD update stage.

## Suggested AMP install flow

1. Put the four template files at the root of a forked AMPTemplates repository.
2. In AMP, go to `Configuration -> Instance Deployment`.
3. Add your configuration repository using `user/repo:branch`.
4. Click Fetch, refresh the browser, then create a new instance from the s&box template.
